01 Server Startup

When the backend starts, main.py is the entry point. It creates the FastAPI app, adds middleware, creates database tables, and mounts all API routes.

$ uvicorn app.main:app --port 8101
main.py
Create FastAPI instance + CORS middleware
app = FastAPI(title="AI Workflow Platform")
app.add_middleware(CORSMiddleware, allow_origins=settings.cors_origins)
↓
db/session.py
Create SQLAlchemy engine + session factory
engine = create_engine(settings.sqlalchemy_database_url)
SessionLocal = sessionmaker(bind=engine)
↓
main.py → startup
Auto-create tables: workflows, documents
Base.metadata.create_all(bind=engine)
# Creates tables from models/workflow.py + models/document.py
↓
api/router.py
Mount all route modules
api_router.include_router(health_router)    # /health
api_router.include_router(auth_router)      # /auth
api_router.include_router(workflows_router) # /workflows
api_router.include_router(documents_router) # /documents
✅ Server listening on port 8101 — Swagger docs at /docs

02 Authentication (JWT Login)

User sends email + password. Server verifies credentials, creates a JWT token signed with a secret key, and returns it. Frontend stores the token in localStorage.

POST /api/v1/auth/token → { username, password }
api/auth.py
Verify username + password against settings
if form_data.username != settings.demo_user_email 
   or form_data.password != settings.demo_user_password:
    raise HTTPException(401, "Incorrect username or password")
↓
core/security.py
Create JWT with email + 60-min expiry
payload = {"sub": "admin@example.com", "exp": now + 60min}
token = jwt.encode(payload, JWT_SECRET, algorithm="HS256")
↓
app.js (frontend)
Store token → use in all future API calls
setToken(tokenData.access_token);
// Stored in localStorage
// Every request: Authorization: Bearer eyJhbG...
🔐 User authenticated — token valid for 60 minutes

03 Endpoint Protection (Every Request)

Every protected endpoint uses FastAPI's Depends(get_current_user_email). This automatically runs before the endpoint logic — extracting and validating the JWT.

Any request with Authorization: Bearer <token>
api/deps.py
Extract token → decode → return user email
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")

def get_current_user_email(token = Depends(oauth2_scheme)):
    email = decode_access_token(token)  # → security.py
    if not email:
        raise HTTPException(401, "Invalid authentication token")
    return email  # "admin@example.com"
✅ Valid → endpoint runs   |   ❌ Invalid/Expired → 401 Unauthorized

04 Document Upload + Async Processing

This is the core flow. Upload triggers: file storage → DB record → Celery task. The worker then reads the file, extracts text, and indexes it into OpenSearch — all in the background.

POST /api/v1/documents/upload → FormData(file)
Phase 1: API (synchronous — user waits)
api/documents.py
Save file to S3
storage = StorageService()
object_key = storage.save_bytes(file.filename, content)
# → S3: s3.put_object(Bucket, Key, Body)
# → Fallback: write to /app/data/documents/
↓
api/documents.py
Create Document record (status: "uploaded")
document = Document(
    filename="contract.pdf",
    object_key="uuid-contract.pdf",
    status="uploaded",    # ← initial status
    workflow_id=workflow_id,
)
db.add(document); db.commit()
↓
api/documents.py
Dispatch Celery task → returns immediately
task = parse_and_index_document.delay(document.id)
# Task pushed to Redis queue
# API returns 201 without waiting for processing
return {document: {...}, task_id: task.id}
⚡ API returns 201 to user — background processing begins
Phase 2: Celery Worker (asynchronous — user doesn't wait)
tasks/document_tasks.py
Update status → "indexing"
document = db.get(Document, document_id)
document.status = "indexing"
db.commit()
↓
tasks/document_tasks.py
Read file from S3 → extract text
raw = storage.read_bytes(document.object_key)
text = raw.decode("utf-8", errors="ignore")
document.extracted_text = text[:200000]
↓
services/search.py
Index into OpenSearch
search.index_document(
    document_id=doc.id,
    filename=doc.filename,
    content=doc.extracted_text,
)
# OpenSearch stores: {document_id, filename, content}
↓
tasks/document_tasks.py
Update status → "indexed" ✅
document.status = "indexed"  # or "index_failed" / "failed"
db.commit()
return {"status": "indexed", "document_id": doc.id}
Document Status Lifecycle:
uploaded → indexing → indexed ✅ / failed ❌

05 Search (OpenSearch + DB Fallback)

User types a query → OpenSearch returns ranked results with highlighted snippets. If OpenSearch isn't configured, falls back to PostgreSQL ILIKE.

GET /api/v1/documents/search/query?query=liability
services/search.py
OpenSearch multi_match query
query: {
  multi_match: {
    query: "liability",
    fields: ["filename^2", "content"]  // filename boosted 2x
  }
},
highlight: { fields: { content: {} } } // returns matching snippets
↓
api/documents.py (fallback)
If OpenSearch unavailable → PostgreSQL text search
Document.extracted_text.ilike(f"%{query}%")
# Basic but works for local development
🔍 Returns: [{filename, score, snippet}, ...] ranked by relevance
